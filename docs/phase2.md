# Phase 2: Active Validation (Google Ads + Landing Page)

Phase 1 measures passive signal — are people talking about this problem? Phase 2 measures active intent — are people searching for a solution and willing to give you their email?

**Trigger:** Run Phase 2 when Phase 1 returns `Market Signal = strong` or `Probability ≥ 0.10`.

**Budget:** $50–100 per project ($10–15/day for 5–7 days).

**Kill signal:** <5 signups after $100 spend → kill. 3+ pre-orders at $99 → build.

---

## Input from Notion

All inputs are already in the Projects table:

| Notion Field | Used For |
|---|---|
| `Project` | Campaign name, page title |
| `Description` | Landing page hero copy |
| `Pain/Desire` | Ad copy angle, keyword themes |
| `Validation Query` | Google Ads keyword seeds |
| `Price/Customer/yr ($)` | Pricing anchor on landing page |
| `TAM Customers` | Audience sizing for ad targeting |

## Steps

### 1. Landing Page

One-pager deployed to GitHub Pages (`jvalansi.github.io/validate-<project-slug>`):

- **Hero:** Problem statement (from `Pain/Desire`) + one-line solution (from `Description`)
- **CTA:** "Join the beta" → inline form collecting email, monthly spend, role
- **Social proof:** Validation signals (HN posts, Reddit threads, existing competitors)

Signup form submits to a shared Google Form → responses stored in Google Sheet `1UO2fp_kUUj2Go8VZ6nJtdoYhzJDAvrUIVM0Wvp5fg_Y` (tab: `Form Responses 1`).

### 2. Google Ads Campaign

Structure:
- **Campaign type:** Search only (disable Display Network expansion)
- **Daily budget:** $10–15
- **Max CPC:** $3–5 (B2B/developer keywords are competitive)
- **Keywords:** Derived from `Validation Query` and `Pain/Desire` — problem-framing terms, not product terms
- **Ad copy:** Generated by Claude and written to the campaign spreadsheet for manual import

### 3. Measurement (Days 1–7)

Daily Slack report to `#proj-project-validation` via cron at 8:47 AM:

| Metric | Good signal | Kill signal |
|---|---|---|
| CTR | >2% | <0.5% |
| CPC | <$3 | >$10 |
| Signups | >1/day | 0 after day 3 |
| Spend per signup | <$20 | >$50 |

### 4. Outreach (Day 5 — auto-triggered)

Claude generates personalised outreach drafts per signup, posted as a Slack thread for manual review and sending:
- "What's your monthly spend on [problem]?"
- "Would you pay $[Price/Customer/yr ÷ 12]/mo for [solution]?"
- Offer a live demo → ask for $99 pre-order

### 5. Decision (Day 7 — auto-triggered)

| Result | Verdict | Notion status |
|---|---|---|
| ≥3 signups at/above price | build | `building` |
| ≥5 signups, <3 strong | validate more | `validating` |
| <5 signups | kill | `killed` |

Move to the next project in ROI order from the Notion Projects table.

## CLI

```bash
# One-time setup (installs cron, checks credentials)
bash phase2/setup.sh

# Launch a new campaign (deploys landing page, generates ads config, registers campaign)
python -m phase2.cli <notion-page-id> [--days 7] [--dry-run]

# Run daily monitor manually
python -m phase2.cli monitor [--dry-run]

# Generate outreach drafts for existing signups (auto-runs on day 5)
python -m phase2.cli <notion-page-id> --outreach [--dry-run]

# Run day-7 kill/build decision (auto-runs on day 7)
python -m phase2.cli <notion-page-id> --decide [--dry-run]
```
